01 — How it works

Set up once. Runs itself.

Four phases. One command to install, one cron schedule to run. Resumable installer means a failed step just means re-running — you don't start over.

  1. 01Install

    bash setup.sh provisions a Cloudflare Pages project, a D1 database, an R2 bucket, applies the schema, pushes secrets, and deploys. Every step writes to .setup-state; if any step fails (network, auth, quota), just re-run and it resumes from the failure.

  2. 02Generate

    A scheduled Cloudflare Worker calls a four-step chain — pick topic → write text → render hero image → publish — with each step idempotent and short enough to fit Cloudflare's request budget. Survives Pages Functions' aggressive isolate kills.

  3. 03Distribute

    New posts appear in /sitemap.xml immediately. IndexNow pings Bing, Yandex and Seznam on every publish. Google picks them up via the sitemap. The RSS feed exposes posts to readers and aggregators.

  4. 04Iterate

    The admin dashboard tracks your queue, published pages, jobs in flight, and per-provider statistics. A dry-run endpoint lets you preview AI output (with full HTML rendering) before it ever touches your database.

03 — Architecture

The stack.

Every component runs on Cloudflare except the LLM call. That call is your choice — Workers AI, OpenAI, Anthropic Claude, Google Gemini, or any of five more.

→

Cloudflare Pages

Serves the public site, blog, programmatic pages, sitemap, robots.txt and the admin SPA. File-based routing through functions/.

→

D1 (SQLite at the edge)

Stores generated posts, programmatic pages, the keyword queue, job state, and an audit log. Six tables; ~50 lines of SQL.

→

R2 (object storage)

Hero images. Served via a Pages Function with immutable cache headers — Cloudflare's CDN handles repeat fetches.

→

Workers AI

Default text + image generation. Llama 3.3 70B for text, Flux 1 schnell for images. Free tier covers most use cases.

→

Scheduled Worker

Separate Worker that calls the admin API on cron triggers. Daily blog at 08:00 UTC, retries at 10/14/18:00, programmatic batch at 09:00.

→

Optional LLM keys

OpenAI · Anthropic · Gemini · Groq · DeepSeek · Mistral · Together · Cerebras. Drop in a key and it joins the fallback chain.

05 — Install

One command. Resumable.

The setup script auto-installs wrangler and runs wrangler login for you if needed. If any step fails, just re-run — it picks up where it died.

# 1. clone the repo
git clone https://github.com/Benjamin-Bloch/pages-seo

# 2. run any flavour
bash setup.sh
# python3 setup.py · node setup.js

# 3. open the admin panel
open https://<your>.pages.dev/admin

07 — FAQ

Questions.

I don't know what I'm doing — what do I do?

Open /ai-setup, click "Copy prompt", and paste it into ChatGPT, Claude or Gemini. The AI gets a self-contained briefing for the whole install + every known failure mode, and walks you through each step one at a time. You report progress, it reacts. Free, takes about 5 minutes.

How much does it actually cost to run?

On Cloudflare's free tier with Workers AI: $0/month. Workers AI gives you 10,000 Neurons/day free — a full-length post (Qwen3 text + a Flux hero image) costs roughly 150–230 Neurons, so the free tier covers ~40–60 long posts per day (more if they're shorter). The default schedule is one blog post plus ten programmatic pages a day, which uses well under 20% of that. The other limits that matter: 100k Pages/Workers requests/day and 5 GB D1 storage. Exceed any of them and you're on Workers Paid ($5/month) plus a few cents per million extra operations.

Do I have to use Workers AI?

No. It's the default because the free tier makes it the lowest-friction option. You can route everything through OpenAI, Anthropic Claude, Google Gemini, Groq, DeepSeek, Mistral, Together AI, or Cerebras. Adding a tenth provider is one entry in functions/_lib/ai.js.

How good is the generated content?

Workers AI (Llama 3.3 70B) produces SEO-coherent, structurally clean copy — title, meta, slug, keywords, H2/H3, FAQ, CTA all in place. The prose is generic. For higher-quality output, plug in Claude or GPT-5 — same toolkit, same workflow, just different output.

Will search engines penalise AI content?

Google's published position is that AI content is fine if it's useful. The toolkit ships with Article + Breadcrumb schema, structured headings, FAQ blocks, primary/secondary keyword targeting, IndexNow pings, canonical URLs. Quality is on the model you choose.

Can I edit posts manually?

Yes. Posts live in your D1 database — edit via the admin UI's post detail view, or run SQL directly with wrangler d1 execute. Hero images live in R2; replace by uploading a new file with the same key.

Is the admin dashboard secure?

Bearer-token auth with a 64-char hex token generated at install. Stored as a Cloudflare Pages secret on the server, in localStorage on your machine. No accounts, no sessions, no password reset flow. Rotate the token via wrangler pages secret put ADMIN_TOKEN if it leaks.
